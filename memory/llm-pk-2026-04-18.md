# LLM PK Report

更新时间：2026-04-18

## 背景

- 目标：用**当前生产中的 DeepSeek Prompt 路径**，对不同厂商/模型做 live PK。
- 关注点：
  - 生成 `5` 道菜时的性能
  - 内容完整度
  - 内容严谨性
  - 在**无缓存命中**情况下的稳定性
- 测试方式：
  - 直接调用厂商 OpenAI-compatible API
  - 绕过服务缓存与推荐主链路
  - 使用 [MealGenerationModelPkLiveTest.java](/Users/zhaoqiang/Documents/Project/what-to-eat/backend/src/test/java/com/quickstart/template/platform/provider/recipeai/MealGenerationModelPkLiveTest.java)

## 固定测试条件

- Prompt：
  - `MealGenerationPrompts.systemPrompt(request.getCuisine())`
  - `MealGenerationPrompts.userPrompt(request)`
- 请求：
  - `sourceText = 今天晚餐想吃家常下饭菜，口味偏鲜香微辣，荤素搭配，适合 2 到 3 人`
  - `dishCount = 5`
  - `totalCalories = 1800`
  - `staple = RICE`
  - `locale = zh-CN`
  - `cuisine = CHINESE`
- Timeout：
  - Qwen：`150s`
  - DeepSeek：测试代码默认 `120s`

## 口径说明

- `deepseek-chat` / `deepseek-reasoner` 都按 DeepSeek 官方当前命名使用。
- `qwen3.6-plus` / `qwen3.6-flash` 使用阿里云兼容模式接口。
- 本轮 Qwen 使用的是**免费试用 API Key**；因此：
  - 内容质量结论可作为方向性参考
  - 时延与稳定性结论对“当前账号、当前额度、当前限流状态”更敏感

## 第一轮：Qwen 3.6 Plus vs DeepSeek Chat（5 轮）

### Qwen 3.6 Plus

- 模型：`qwen3.6-plus`
- 成功率：`2 / 5`
- 失败原因：`3 / 5` 为 `HttpTimeoutException`
- 成功样本平均耗时：`66.0s`
- 平均完整度：`91.0 / 100`
- 平均严谨性：`75.0 / 100`

### DeepSeek Chat

- 模型：`deepseek-chat`
- 成功率：`5 / 5`
- 平均耗时：`90.5s`
- 平均完整度：`92.8 / 100`
- 平均严谨性：`78.8 / 100`

### 结论

- `qwen3.6-plus` 不是质量差，而是**稳定性明显不够**。
- `deepseek-chat` 在当前 5 道菜主链路里更稳，更适合作为线上默认模型。

## 第二轮：DeepSeek Chat vs Qwen 3.6 Flash（3 轮）

### Qwen 3.6 Flash

- 模型：`qwen3.6-flash`
- 成功率：`3 / 3`
- 平均耗时：`50.3s`
- 平均完整度：`93.0 / 100`
- 平均严谨性：`84.7 / 100`

### DeepSeek Chat

- 模型：`deepseek-chat`
- 成功率：`3 / 3`
- 平均耗时：`79.1s`
- 平均完整度：`95.0 / 100`
- 平均严谨性：`78.7 / 100`

### 结论

- `qwen3.6-flash` 是目前测试里**速度最佳**的 Qwen 方案。
- `deepseek-chat` 的菜谱结构完整度仍略高。
- 如果后续要继续评估 Qwen 进入主链路，`flash` 明显比 `plus` 更值得跟进。

## 第三轮：DeepSeek Reasoner vs Qwen 3.6 Plus（3 轮）

### Qwen 3.6 Plus

- 模型：`qwen3.6-plus`
- 成功率：`1 / 3`
- 平均成功耗时：`62.9s`
- 平均完整度：`90.0 / 100`
- 平均严谨性：`83.0 / 100`

### DeepSeek Reasoner

- 模型：`deepseek-reasoner`
- 成功率：`3 / 3`
- 平均耗时：`190.6s`
- 平均完整度：`93.0 / 100`
- 平均严谨性：`74.7 / 100`

### 结论

- `deepseek-reasoner` 稳定，但在当前 5 道菜请求下**时延过高**，不适合作为默认主链路。
- `qwen3.6-plus` 依然暴露出稳定性不足的问题。
- 这组没有产生优于 `deepseek-chat` 或 `qwen3.6-flash` 的工程选择。

## 综合判断

### 当前最稳的默认选择

- **`deepseek-chat`**
- 原因：
  - 成功率最高且稳定
  - 完整度整体最稳
  - 已经与当前产品 Prompt 路径对齐

### 当前最值得继续观察的备选

- **`qwen3.6-flash`**
- 原因：
  - 3 轮成功率 `100%`
  - 明显更快
  - 严谨性分数在这轮样本里更强

### 当前不建议作为默认主链路

- `qwen3.6-plus`
  - 质量不差，但超时率过高
- `deepseek-reasoner`
  - 质量没有显著拉开，但时延过高

## 建议

### 线上默认模型建议

- 继续使用 `deepseek-chat`

### 下一轮值得补的实验

- 使用**正式付费 Qwen Key**再跑一轮 `qwen3.6-flash`
- 追加一个更复杂的真实业务输入：
  - 更长 sourceText
  - 更明确的忌口
  - 更严格的热量约束
- 如果要继续试更强档的 Qwen，再测：
  - `qwen3-max`
